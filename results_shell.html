Overview

This dashboard measures how the prevalence of skills associated with each node in the AI Skill Matrix changes between 2023 and 2026 job postings. The mapping from raw job posting skills to matrix nodes is performed using a semantic similarity approach.

Data Sources

  • Job postings: Lightcast job posting data with pre-extracted standardized skills, filtered by O*NET occupation codes.
  • Skill Matrix: 55 nodes organized into 3 categories (Declining, Enduring, Emerging), each classified as hard or soft skill. The matrix defines a parent-child hierarchy used in the dashboard layout.

Step 1: Building the Skill-to-Node Dictionary (LLM-adjudicated)

The mapping is built in two stages. A candidate-generation stage embeds node names and skills (all-MiniLM-L6-v2, cosine ≥ 0.45, winner-take-all) to define the review universe: every skill it mapped, plus every skill passing the ≥5%-either-year analysis filter (N = 1,170). An adjudication stage then reviews each of those skills individually with a frontier LLM (Claude, Anthropic) under a written protocol (llm_review/PROTOCOL.md): each skill is assigned to the single best-fitting leaf node (parents are aggregates and never receive direct assignments), or left unmapped when no node covers its primary meaning — skills are not force-fitted. Each mapping carries a confidence grade (3 = clear construct containment, 2 = reasonable fit, 1 = borderline).

The adjudication corrected substantial embedding-stage errors: only 38% of the original embedding assignments survived review unchanged, and generic business skills that had leaked into AI-specific nodes were re-assigned to Enduring nodes.

Step 1b: AI-Exposure Scores

Each skill also receives an AI-exposure score (0.05–0.95): the degree to which current generative AI can perform the task implied by the label, scored against anchored rubric levels (0.9 = AI performs end-to-end; 0.7 = AI does most, human validates; 0.5 = mixed; 0.3 = mostly human; 0.1 = human bottleneck), following the task-level exposure logic of Eloundou et al. (2023) and Felten et al. (AIOE). Scoring is matrix-independent: it never conditions on the node a skill maps to. The legacy keyword rubric (neutral 0.5 base ± keyword hits) is retained as an independent robustness measure: Pearson r = 0.61 across all 1,170 skills, r = 0.80 on the 511 skills where the keyword rubric is informative (moves off its 0.5 default).

Step 2: Computing Node Penetration

For each combination of (occupation, year, node), we calculate the penetration rate: the percentage of job postings that contain at least one standardized skill mapped to that node.

  1. Each job posting's comma-separated standardized skills are looked up in the dictionary.
  2. A job is counted as "containing" a node if any of its skills map to that node.
  3. Penetration = (number of jobs containing the node / total jobs in that occupation-year group) × 100.

Step 3: Comparing 2023 vs 2026

The change in penetration (in percentage points) is computed for each node per occupation. The dashboard displays both years side by side, with the 2026 bar colored green if penetration increased and red if it decreased.

Limitations

  • The adjudicated universe covers skills that are either embedding-mapped or above the ≥5% frequency floor. Rare skills outside it are not represented in the penetration metric; their contribution is bounded by their rarity.
  • Each skill maps to exactly one leaf node (the best construct fit). Some skills could reasonably map to multiple nodes; single assignment avoids double counting at the cost of granularity.
  • Mapping and exposure are single-rater LLM adjudications under a written protocol, cross-checked against the embedding and keyword layers — not expert-panel or firm-validated task measurements. All adjudications are stored in reviewable batch files (llm_review/batches/).
  • Exposure is scored on skill labels, not on observed task content within firms.
  • Job volume grew significantly between 2023 and 2026 across all three occupations, which may reflect market trends or data collection changes rather than skill shifts alone.
  • Two time points, three management occupations: correlational evidence, not causal identification.
